JavaScriptの文法とデータ型のポイント

基本

  • JavaScriptはUnicode文字セットを使用する
  • 大文字と小文字を区別する
  • 命令は文 (statement) と呼ばれ、セミコロン (;) によって区切られる

コメント

  • コメントはホワイトスペースのように扱われ、スクリプトの実行から除外される
  • 行コメント // 一行のコメントは先頭に記号を付ける
  • ブロックコメント 
    /*
                        複数行からなる
                        コメントを囲む
                    */
  • ハッシュバンコメント #! /usr/bin/env node

宣言

  • 変数を宣言する方法は3種類: var, let, const

変数

  • 変数はデータを格納し参照するための識別子
  • 文字、アンダースコア (_)、ドル記号 ($) から始まる。続く文字には数字 (0–9) も使用できる

変数の宣言

  • 変数は使用する前に常に宣言する
  • 宣言方法は var（ローカル変数・グローバル変数の両方）と let／const（ブロックスコープのローカル変数）

例: 変数の宣言と初期化


const foo = {
    bar1: 'sample001',
    bar2: 'sample002',
};
// fooからbarプロパティを抽出
const { bar1 } = foo;
// 抽出したbarプロパティの値をコンソールに出力
console.log(bar1);

データ型

  • JavaScriptには以下のデータ型がある
  • プリミティブ型（Boolean, null, undefined, Number, BigInt, String, Symbol）
  • オブジェクト型（Object）

データ型の変換

変数宣言時にデータ型を指定する必要がなく、スクリプト実行時に必要に応じてデータ型が自動的に変換される。

数値と '+' 演算子


const x = "答えは " + 1 + 2;
console.log(x); // "答えは 12"
const y = 1 + 2;
console.log(y); // 3

リテラル

  • 配列リテラル
  • 論理値リテラル
  • 数値リテラル
  • 浮動小数点リテラル
  • オブジェクトリテラル
  • 正規表現リテラル
  • 文字列リテラル

注意点

オブジェクトはプロトタイプのプロパティを持っている。プロトタイプを理解するには少し先の内容です。
